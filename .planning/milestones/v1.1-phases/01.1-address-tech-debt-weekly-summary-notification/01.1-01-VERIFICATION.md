### Deviations from PLAN (Documented in SUMMARY.md)

**1. [Auto-fixed] Init signature split due to Swift actor-isolation constraint**

- **Plan stated:** `init(onDone: @escaping () -> Void, weeklySummaryService: WeeklySummaryService = WeeklySummaryService())`
- **Actual implementation:** Split into two initializers:
  - `init(onDone: @escaping () -> Void, weeklySummaryService: WeeklySummaryService)` (explicit, required parameter)
  - `init(onDone: @escaping () -> Void)` (convenience, delegates to explicit init after constructing default `WeeklySummaryService()` in the body)
- **Reason:** Swift's actor-isolation model checks default *parameter* value expressions in an isolation-agnostic context, even when the enclosing init is `@MainActor`-annotated. Since `WeeklySummaryService.init` is `@MainActor`-isolated, the default-parameter syntax fails to compile. Constructing `WeeklySummaryService()` inside the convenience init's body (which runs on the isolated context) succeeds. This is a necessary fix, not a scope change.
- **Impact on plan acceptance criteria:** One literal grep (`weeklySummaryService: WeeklySummaryService = WeeklySummaryService()`) will not match because the syntax is split, but all other acceptance criteria pass:
  - `grep -Fc "func disableWeeklySummary"` returns exactly 1 ✓
  - `grep -Fc "private let weeklySummaryService: WeeklySummaryService"` returns exactly 1 ✓
  - `grep -Fc "Task { await disableWeeklySummary() }"` returns exactly 1 ✓
  - `xcodebuild build` exits 0 with zero new warnings ✓
- **Verification:** Both `OnboardingView.swift`'s existing call site (`HealthStep(onDone: { finish() })`) and the `#Preview` compile unchanged via the convenience init. Existing tests pass. This is a safe, self-contained refactoring that preserves all observable behavior.
