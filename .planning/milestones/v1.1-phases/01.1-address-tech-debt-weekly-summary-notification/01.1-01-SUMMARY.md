## Decisions Made
- **Deviated from the plan's literal init signature** `init(onDone:weeklySummaryService: WeeklySummaryService = WeeklySummaryService())` because it does not compile: `WeeklySummaryService` is a `@MainActor final class`, and Swift checks default *parameter* value expressions in an isolation-agnostic context regardless of the enclosing function's own isolation (confirmed by testing both an unannotated init and an explicitly `@MainActor`-annotated init — both fail with "call to main actor-isolated initializer ... in a synchronous nonisolated context"). This differs from `WeeklySummaryService.init`'s own `center: NotificationScheduling = WeeklySummaryService.defaultCenter()` pattern, which works because `defaultCenter()` is an explicitly `nonisolated static func` that never touches the actor-isolated class initializer itself — RESEARCH.md/PATTERNS.md's proposed literal syntax did not account for this distinction. Resolved by splitting into two initializers: `init(onDone:weeklySummaryService:)` (no default) plus a convenience `init(onDone:)` that calls `self.init(onDone:weeklySummaryService: WeeklySummaryService())` from inside its own (MainActor-isolated, per the type's `SWIFT_DEFAULT_ACTOR_ISOLATION = MainActor` project setting) body — mirroring how `RootShellView`/`WeeklySummarySection` already construct `WeeklySummaryService()` as plain property initializers (which also run in an isolated context, not as default-argument expressions). Both existing call sites (`OnboardingView.swift`'s `HealthStep(onDone: { finish() })` and the `#Preview`) continue to compile unchanged via the convenience init.

- **Found during:** Task 1 (`xcodebuild build` acceptance check)
- **Issue:** The plan's mandated exact text `weeklySummaryService: WeeklySummaryService = WeeklySummaryService()` as a single init's default parameter value fails to compile (`error: call to main actor-isolated initializer 'init(center:defaults:)' in a synchronous nonisolated context`) because `WeeklySummaryService` is `@MainActor`-isolated and default argument expressions are checked isolation-agnostically in Swift's actor model, independent of the enclosing init's own isolation
