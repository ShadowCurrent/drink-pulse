## Shared Patterns

### Constructor injection over `@Environment` for non-shared services
**Source:** `drinkpulse/Services/WeeklySummaryService.swift:27-33` (own init seam)
**Apply to:** `HealthStep`'s new `weeklySummaryService` stored property/init param.
**Why not `@Environment`:** `HealthServiceEnvironment.swift` documents that
`HealthService` needs `@Environment` specifically because it requires ONE shared
instance across consumers (serialization invariant). `WeeklySummaryService` has no
such requirement — `WeeklySummarySection.swift` line 26 already does
`private let service = WeeklySummaryService()`, its own independent instance. Adding
`@Environment` only to `HealthStep` would create a new asymmetry, not resolve one.
```swift
init(
    center: NotificationScheduling = WeeklySummaryService.defaultCenter(),
    defaults: UserDefaults = .standard
) {
    self.center = center
    self.defaults = defaults
}
```

### Toggle-generation guard (race protection, WR-02)
**Source:** `WeeklySummarySection.swift` `toggleGeneration` (lines 24, 67, 88, 103,
108) and `HealthStep.swift` `weeklySummaryToggleGeneration` (lines 30, 120, 122, 165,
174).
**Apply to:** Preserve as-is — the `cancel()` fix goes in the toggle-off `else`
branch, which today does NOT touch `weeklySummaryToggleGeneration` (only the
`if newValue` branch increments/checks it). RESEARCH.md and CONTEXT.md both confirm:
do not add generation-guard logic to the new `disableWeeklySummary()` call unless a
concrete race is found — none was identified.

### FakeNotificationCenter (test double)
**Source:** `drinkpulseTests/Services/ReminderServiceTests.swift:10` — declared with
no access modifier (internal), tracks `removedBatches: [[String]]`.
**Apply to:** `HealthStepTests.swift` — reuse directly via `@testable import
drinkpulse`, do not declare a new fake (per D-03's explicit "reuse that pattern
rather than inventing a new one").
